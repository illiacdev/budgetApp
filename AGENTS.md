# 프로젝트 규칙

## 프로젝트 설명
- CSV 기반 가계부 웹 앱.
- 사용자는 CSV 파일을 업로드하고, 일별 및 월별 통계를 웹 화면에서 확인한다.
- 대용량 CSV도 안정적으로 처리해야 한다.

## 코딩 규칙
- 타입 힌트는 필수.
- 함수 하나는 50줄을 넘기지 않는다.
- 순환복잡도는 10 이하를 유지한다.
- 가능한 한 순수 함수로 분리한다.

## TDD 규칙
- 구현보다 테스트를 먼저 작성한다.
- 실패하는 테스트를 확인한 뒤 최소 구현을 추가한다.
- 리팩토링은 테스트가 통과한 상태에서만 진행한다.

## 품질 규칙
- 브랜치 커버리지를 포함해 테스트 커버리지 100%를 목표로 한다.
- 정적분석을 반드시 실행한다.
- GitHub Actions에서 동일한 검사를 수행한다.

## QA 규칙
- 각 Phase 종료 시 QA 서브에이전트로 검토한다.
- 검토 항목: 테스트 누락, 복잡도 초과, 대용량 처리 위험, CI 누락.

## 추천 검사 명령
- `pytest`
- `pytest --cov=app --cov-branch`
- `ruff check .`
- `radon cc -s -a app`
- `xenon --max-absolute B --max-average A app`
