# 프로젝트 규칙

## 프로젝트 설명
- 이 저장소는 Python CLI 가계부 앱 프로젝트다.
- CSV 기반 거래 데이터를 읽고, 지출/수입 내역을 조회하고, 요약 통계를 제공한다.

## 코딩 규칙
- 타입 힌트는 필수다.
- 함수 하나는 최대 50줄을 넘기지 않는다.

## TDD 규칙
- 구현 전에 반드시 테스트를 먼저 작성한다.
- 테스트가 실패하는 상태를 확인한 뒤 최소 구현을 추가한다.

## 품질 규칙
- 순환복잡도는 10 이하를 유지한다.

## 품질 검수 규칙
- 커밋 전에 반드시 `qa_engineer` 서브에이전트로 품질 검수를 수행한다.
- 검수 항목은 테스트 누락, 엣지 케이스, 순환복잡도, 타입 힌트, 리팩토링 필요성이다.

## 테스트 실행 명령어
- `pytest`
- `radon cc`

## 커밋 규칙
- 하나의 기능이 개발되면 커밋하고 푸시한다.
